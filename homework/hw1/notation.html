Sorting Algorithms

Computational Complexity Notation

Sorting Algorithms are judged by various factors, such as time complexity, memory usage, and stability. One of the most common ways of comparing their efficiency is using Big O Notation.
Big O Notation describes the trends of an algorithm as its input size increases. For example, imagine a list of values that are not currently in order:
5 3 1 6 7 2 4 8

A very simple sorting algorithm, like bubble sort, works by going left-to-right, comparing two values at a time, and swaps them if the right value is smaller than the left.
The algorithm makes its way through the list multiple times, starting from the first value and continuing until the last, until the entire list is sorted.
At the worst case scenario, the algorithm will have to revisit each item for as many items there are in the list.

Image Source

If the number of items on a list is represented by n, then in the worst-case scenario this sorting algorithm will have to visit n number of items for n number of times, or n2 number of times.
The overall time complexity of this algorithm can be described as O(n2). The time complexity measures how much time an algorithm will rougly take proportional to its input size n.
O() is the formal notation for this measurement, and inside the parantheses is the complexity represented in a function.
Big O Notation can be used to measure the best-case, average-case, or worst-case scenario of an algorithm, but in Computer Science it is often more useful to discuss algorithms by their worst-case scenario performance, especially when comparing them against one another.


Space Complexity

Similarly to how time performance can be measured using Big O Notation, memory usage can also be analyzed with the same metric.
An algorithms memory efficiency, or space complexity, is the measurement of how much storage or RAM may be allocated in relation to the input size.
The previously mentioned Bubble Sort, for example, relies on no extra memory usage for the comparisons.
This means that, no matter how big the input size is, the memory usage is functionally the exact same, giving it a space complexity of O(1).

Space-Time Tradeoff

An interesting pattern to note is that there is often an inverse relationship between Time Complexity and Space Complexity - meaning that the faster an algorithm is, the more storage it typically uses. This is because some of the fastest accessing and sorting algorithms rely on saving quick lookup information.
This is not a unique feature of sorting algorithms, but is a very common design principle of computer science overall.
The pattern of sacrificing storage use for response time - or vice versa - appears in operating system design, databases, networking, and general programming.
For a more in-depth look at various types of tradeoffs, refer to the following GeeksForGeeks entry: Time-Space Trade-Off in Algorithms.
